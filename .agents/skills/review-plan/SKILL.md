# Purpose

Use this skill when a user asks you to review, critique, or improve a plan before implementation.

# Trigger Words/Phrases

- review the plan
- review plan
- critique this plan
- improve this plan
- find gaps in this plan
- identify blind spots
- sanity check this plan

# Instructions

1. Read the full plan carefully.
2. Identify issues in these categories:
   - Unclear goals or success criteria
   - Missing assumptions
   - Ambiguous terminology
   - Missing requirements
   - Edge cases not addressed
   - Technical risks
   - Operational risks
   - Testing gaps
   - Rollout or migration gaps
   - Ownership or sequencing issues
3. For each issue, explain:
   - What is unclear or risky
   - Why it matters
   - A concrete suggestion to fix it
4. Apply the suggestions by rewriting the plan.
5. Keep the revised plan faithful to the original intent. Do not introduce unnecessary scope.
6. If the plan is too vague to safely rewrite, make reasonable assumptions and list them before the revised plan.
